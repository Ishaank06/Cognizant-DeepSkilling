# Exercise: JPA, Hibernate, and Spring Data JPA Comparison

## 1. Core Differences

| Feature | Java Persistence API (JPA) | Hibernate | Spring Data JPA |
| :--- | :--- | :--- | :--- |
| **What is it?** | A Java EE/Jakarta EE specification (JSR 338). | A fully-featured ORM (Object-Relational Mapping) framework. | An abstraction layer on top of a JPA provider. |
| **Role** | Defines standard rules, annotations (like `@Entity`, `@Id`), and interfaces. | Implements the JPA specifications and provides concrete database operations. | Reduces boilerplate repository implementation code. |
| **Abstraction Level** | Specification Level (No concrete implementation). | Implementation Level (Direct ORM implementation). | High-Level Abstraction (Abstracts away Session/EntityManager calls). |
| **Transaction Management** | Requires manual or container-managed configurations. | Managed via transaction API block (`session.beginTransaction()`). | Automated via declarative annotations like `@Transactional`. |

## 2. Code Comparison

### Native Hibernate Approach (Manual Session & Transaction Management)
In the native Hibernate model, developers must manually open sessions, start transactions, handle exceptions, perform rollbacks, and close sessions.

```java
/* Method to CREATE an employee in the database */
public Integer addEmployee(Employee employee){
    Session session = factory.openSession();
    Transaction tx = null;
    Integer employeeID = null;
    try {
        tx = session.beginTransaction();
        employeeID = (Integer) session.save(employee);
        tx.commit();
    } catch (HibernateException e) {
        if (tx != null) tx.rollback();
        e.printStackTrace();
    } finally {
        session.close();
    }
    return employeeID;
}
```

### Spring Data JPA Approach (Declarative & Abstraction Layer)
With Spring Data JPA, boilerplate session management, transactions, and SQL statements are abstracted. You only define repository interfaces and apply declarative transactions.

#### Repository Layer: `EmployeeRepository.java`
```java
public interface EmployeeRepository extends JpaRepository<Employee, Integer> {
}
```

#### Service Layer: `EmployeeService.java`
```java
@Autowired
private EmployeeRepository employeeRepository;

@Transactional
public void addEmployee(Employee employee) {
    employeeRepository.save(employee);
}
```

## 3. Hands-on Codebase Directory

This project contains a fully operational Spring Boot demonstration showing both approaches in a unified app:

*   **`pom.xml`**: Bootstraps the application with `spring-boot-starter-data-jpa` and `h2` database.
*   **`Employee.java`**: The JPA Entity representing the `employees` table.
*   **`EmployeeRepository.java`**: The Spring Data JPA repository.
*   **`EmployeeService.java`**: The service performing declarative database saves.
*   **`HibernateEmployeeService.java`**: The service performing manual Hibernate Session transaction management.
*   **`EmployeeManagementApplication.java`**: Main startup class executing both approaches side-by-side.

---

## 4. How to Run the Demonstration

Run the application using the following Maven command:

```bash
mvn compile spring-boot:run
```

*Note: If you run into SSL/Certificate issues in restricted network environments, you can bypass certificate validation using:*
```bash
mvn -Dmaven.wagon.http.ssl.insecure=true -Dmaven.wagon.http.ssl.allowall=true compile spring-boot:run
```
